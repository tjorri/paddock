# Security Policy

## Reporting a vulnerability

**Please do not file public GitHub issues for security problems.** Public issues are visible to everyone and can put users at risk before a fix is available.

Instead, report vulnerabilities privately through GitHub's security advisories:

- **<https://github.com/tjorri/paddock/security/advisories/new>**

This opens a private channel between you and the maintainers. We can discuss the issue, coordinate a fix, and (with your consent) credit you in the advisory when we publish it.

## What to include

If you can, please include:

- A description of the vulnerability and its impact.
- Steps to reproduce, or a proof-of-concept.
- The affected version(s), commit SHA, or branch.
- Any suggested mitigation, if you have one in mind.

## Scope

Paddock is pre-1.0 and the security surface is still evolving. Reports are welcome against any component in this repository — controller, webhook, broker, proxy, CLI, collector, reference harnesses, Helm charts, and CI workflows.

If you are unsure whether something qualifies as a security issue, err on the side of reporting it through the advisories link above and we will triage from there.
