Verify the controller and broker rolled out (skip if you used `helm install --wait`):

  kubectl -n {{ .Release.Namespace }} rollout status deploy/paddock-controller-manager
  kubectl -n {{ .Release.Namespace }} rollout status deploy/paddock-broker

For the canonical end-to-end walkthrough — local Kind cluster, both
reference harnesses, BrokerPolicy setup, and the observability surface
— see the quickstart:

  https://github.com/tjorri/paddock/blob/main/docs/getting-started/quickstart.md

The CLI plugin (`kubectl paddock`) is built from this repo via `make cli`
and gives you `submit`, `status`, `events`, `logs`, `audit`, and
`policy` subcommands. See `kubectl paddock --help` once it's on PATH.
