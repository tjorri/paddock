# Reference

Lookup-oriented content. Most of this is autogenerated from source — do
not hand-edit autogenerated files; regenerate them via the relevant `make`
target.

To be written or generated:

- `crds/` — autogenerated from kubebuilder type comments. One file per
  Kind plus an index.
- `cli/` — autogenerated per-command CLI documentation.
- `configuration.md` — controller and broker flags + environment
  variables.
- `audit-events.md` — schema for emitted audit events.
- `glossary.md` — term list.

Hand-written reference rots inside one release. If you add a hand-written
reference page here, plan its replacement by autogeneration.
