## 1. Threat-coverage matrix

The matrix maps each threat actor (T-N from `threat-model.md` §3) crossed with a STRIDE category to the existing test surface. Cells contain test names + status. `✓ covered` = at least one test asserts the defence. `⚠ partial` = some shape covered but a documented edge case is missing. `× gap → TG-NN` = no test exercises this defence; the gap entry in §3 specifies the scenario.

Test-name conventions:
- Unit: file::TestName, e.g., `proxy/server_test.go::TestProxy_DeniesWhenHostNotInAllowList`.
- Integration: envtest-suite test, e.g., `controller/suite_test.go` (Ginkgo `Describe/It`).
- E2E: scenario name from `test/e2e/e2e_test.go`, e.g., `e2e v0.3 hostile prompt egress-block`.
